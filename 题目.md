## 基本知识

什么是数据分析：

数据分析让工作开展有据可依：<span style="border-bottom:1px solid lightblue;">以往发生了什么、为什么发生、如何做、将来业务走势</span>

数据分析的意义：让数据分析与实际业务相关联，从中提取洞见，为企业发展提供参考，以减少未来的不可预料

数据分析是一套工作流程，而不是单纯地利用工具

数据分析在电商中的应用：用户洞察、精准营销、库存管理、未来预测

- 用户画像：年龄、性别、所属行业、常用网络、所用终端、业务类别、使用时间和位置、场景等

- 精准营销：精准划分用户，准确定位产品，提升营销精准度

- 库存管理：解决库存积压和库存不足，提升销售预判能力、物资调配能力

- 未来预测：根据现有已发生数据，开展月度、季度、年度等多维度数据分析，利用同比、环比等方式，预测未来业务数据，协助企业及时调整人员配比、调整业务方案，利于企业提前做出合理化决策

数据分析实施步骤：提出问题或需求 -> 理解数据 -> 数据清洗 -> 构建模型 -> 构建数据可视化

- 提出问题或需求

需求的关联方，如用户方、商城方、销售方

用户方的需求点，例如省采购数据及采购进度、各下属平台公司的采购表现

商城方的需求点，例如交易量及业务利润、各省下单数据及占比、明年预估数据

销售方的需求点，销售数据

- 理解数据

理解数据包含两个方面：① 采集数据集 ② 查看数据集(了解每个字段的意义和数据集的描述性统计信息)

- 数据清洗

即是数据的预处理

数据清洗大致分为 7 个步骤：选择子集、列名重命名、删除重复值、缺失值处理、一致化处理、数据排序、异常值处理

- 构建模型

得出问题答案

- 构建数据可视化

数据分析是一种决策的方法,是从历史数据总结经验并推演出结论的完整分析过程。有了数据记录，就能够总结过去，持续优化。

利用大数据从整体结构上提升核心竞争力以及运营工作效率。

数据分析师是干嘛的：

解决数据分析的决策流程在推进的过程中遇到的问题，比如没有数据、数据杂乱、数据获取困难、缺少数理统计知识等。

数据分析师的工作：

获取数据 - 处理数据 - 分析数据

取数、对口径、底层重构

高价值的工作：

- 提出好问题
- 给出好答案
- 总结分析套路
- 提供高效工具
- 解决困难问题

数据分析思维

分析问题三步骤：明确问题 -> 分析问题 -> 解决问题

明确问题：问题是现实和预期之间的差距 -> 现实、预期、差距

- 现状有问题 - 数据准确性存在问题
- 预期有问题 - 业务周期性波动问题
- 现实和预期真的存在差距 - 真正需要深挖原因的问题

分析问题：突发性问题或持续性问题

解决持续性问题 -> 维度拆解（细化问题的颗粒度 -> 将大问题分解成小问题）

关于利润

企业利润 = (产品价格 - 单位成本) x 产品销量

利润来源的核心是成本是合理的

产品价格的上限是“客户因为某种需求被满足而愿意支付的价格” -> 马斯洛需求层次理论

## 技术题

### 题目：某电商 12 月支付用户数环比上月下降 x%，请问要如何分析找出原因？

三个步骤：

1. 明确问题
2. 分析背后的逻辑
3. 根据分析结果提出建议

#### 明确问题

确定数据提取、计算过程都是正确的

是否是季节性的原因？淡旺季？或者上月是有大型活动，例如双十一，618，过年等；这个月没有大型活动。国家政策、天气等宏观和不可控原因。

能否缩小问题的范围，是某一个地区有问题，还是所有的地区都有这个问题；是只有这个月有问题，还是前面几个月都是下降的趋势。

关注的指标：支付用户数量

支付用户数量 = 用户数量 \* 转化率

是用户数量下降了，还是转化率下降了？亦是两者都下降？

分析思路：

```
支付用户数量
    用户数
        新用户
            渠道A
            渠道B
            渠道C
        老用户
            渠道A
            渠道B
            渠道C
    转化率
        渠道A转化率
        渠道B转化率
        渠道C转化率
```

三种假设：

假设 1：新用户的数量减少

    假设渠道A和渠道B获得的新用户数量不变，渠道C获得的新用户数量下降

    对渠道C进一步来拆分为：年龄、地区、职业、性别等人口特征。

    利用对新用户进行机器学习（例如K-Mean）或者一些分析模型（例如RFM）划分的结果分析某个群体在该渠道减少的情况

假设 2：老用户的数量减少

    可以利用上述方法对老用户进行分析

    也可以从产品体验、竞品分析、用户反馈等维度来分析原因；这里可以看用户投诉情况、客户沟通情况、竞品近期有无活动等情况

假设 3：转发率降低

可以使用**漏斗模型**来找出哪一环节的流失率比较大，通常用户完成支付的流程为：

去往商品页面（广告点击、直接搜索等方式） –> 商品浏览 –> 加入购物车 –> 完成支付

假设第 1-2 步骤流失率很大，那么可能是广告投放效果不好，商品标题用户搜索不到等原因

假设第 2-3 步骤流失率很大，那么可能是商品价格贵，有竞品价格更加优惠，标题和实物不相符，客户不在线，好评少等原因

假设第 3-4 步骤流失率很大，那么可能是用户忘记付款，付款方式少，有 bug 等原因。

### 我有 1 亿注册用户，月活跃 2500 万，有什么办法把剩下 7500 万非活跃的变成活跃的

要有**质疑精神**，但不能直接怼回去，数据分析师永远不要做判官，我们只提供分析方法，老板才是判官

1. 确认问题：拿出一张纸画出结构

问面试官：

您好，确认一下您的问题，看我理解的对不对，就是咱们现在有一亿的注册用户，活跃的用户是 2500 万，在活跃用户以外还有很大的空间，我们想扩大活跃空间，对不对？

目的：

我们不了解公司情况，对方又必须我们回答，我们要把节奏拖下来，为我们自己争取更多的说话机会，而且重复问题是一种很专业的表现（尊重对方、专业性高）

2. 统一口径

问面试官：

您说有 2500 万的活跃用户，那怎么样算是活跃？是打开一次 app，还是使用一次产品？或者是消费一次呢？

有一亿的注册用户，是有一亿个客户表格信息？还是有一亿个系统记录 ID，再或者是开通了你们某项功能呢？

目的：

统一口径、写下来显得你尊重对方、箭头暗示后面还有下文，让其不会觉得我们是在拖延时间

3. 询问业务

画出结构图

问面试官：

那么我们是只考虑用户的活跃率问题，考不考虑用户的付费问题呢？

目的：让面试官多和你聊公司业务相关问题

4. 给出结论

1.既然有 2500 万 的活跃用户，那么我们可以根据这个活跃用户构建出**用户结构**，梳理出活跃**用户画像**，可以对比我们活跃用户的来源渠道，付费习惯等。

2.构建了用户结构之后，我们可以将不同类用户的生命周期刻画出来，然后发现各类用户是在什么节点增长、流失的，可以在相应的节点作出一些动作。

技巧：

这一招叫反客为主，是应付不太清楚背景，又必须回答问题的情况下秘籍，关键在于降低对方速度，给自己缓冲，留下思考时间，拖进我们的节奏

同时，可视化可以展示我们的能力与信心

并且，即使这个问题我们不太清楚答案，一个流程走下来，也显得 80%环节是清晰的！这能大大缓解提问者的焦虑，为我们挣得主动

注：

用户画像

是什么：将用户的每个具体信息抽象成标签，利用这些标签将用户形象具体化，从而为用户提供有针对性服务的工具（一种了解用户的方式）；

有什么用：自动推送，促进消费

### 客户价值指标定义问题

数据分析三大要素：数据来源、指标口径、统计方式 （如果给的数都是错的，分析有啥用？）

问题:

- 领导问: 我需要建一个指标叫，顾客价值分级

- 你问领导: 这个指标怎么建?用来干什么?为什么想建?

- 领导说: 我不知道怎么建; 暂时没啥用; 我就觉得 CRM 里该有一个字段

总之，自己想!

解题思路：

先将字段分解，然后给出选项，并且从易到难排序，先推简单的，再推难的

```

客户价值分类
    客户
        现实客户
        潜在客户
    价值
        KPI价值
        实际利润
        综合价值
        潜在价值
    分类
        二八法则
        十分位法
        综合评估
```

主要考点：

如何自己建立指标

工作思路

1. 审题：先审题，不要一看到问题就想着上模型

2. 确认口径：如何计算这个问题，确保口径正确

3. 分解问题：如果问题包含好几个名词或者指标，需要做分解

4. 给选项

在面试特别重要，多给几个选项，优先推荐其中一个，给自己留下后路

越是主动给选项，越容易控制领导期望

5. 找领导确认：不要自己埋头干，多和领导确认沟通

6. 分步骤完成：落地逐步做，简单的先做，逐步给领导展现成果

### 销量下降问题

某外卖平台，HR 问：“我们在上海地区的交易额比较差，分析下原因”

这是一个典型的 为什么 的问题，在回答此类问题时，答案一定是原因 1 可能是、原因 2 可能是、原因 3 可能是……（直面回答）

```

上海交易额差
    是什么
        差 <--> 好
            和谁比
            多差算差
        时间、地点、区域
            何时、区域、认识来源
    为什么
        对比标杆
        产品
        用户
        商户
        竞品
    做什么
```

审题：

比较差 是一个形容词，那么就要问比较差的标杆是谁？多差算差？好的标准是什么？

是什么：

在什么时间比的，是这一天差了，还是一周差了，还是一直以来都很差？（一次性/还是连续性？，持续性/间歇性？）

一次性可能是事件导致的，间歇周期性要考虑周期问题，最好具体到什么时间、什么情况、什么地方发生了什么事情？

问题是越来越严重还是有向好的趋势？

为什么：

给的原因最好是可以对应到部门，比如用户、产品、竞品等，不要给什么用户活跃低了，因为用户活跃低了好像和每个部门有关

分析问题三大问：

1. 是什么？（确认数据）

2. 为什么？（寻找原因）

3. 做什么？（给出建议）

这是个典型的“是什么+为什么”问题，是什么被隐藏掉了，很多人容易掉坑！

参考答案：

1. 明确数据的来源和准确性：是哪个部门说业绩变差的？他们的数据来源是什么？是否核对的数据确保数据没有问题?

2. 明确数据口径和时间段，哪个时间段内的业绩，对标是什么，评价标准是什么，为什么说变差了?

3. 明确这是不是个问题，业绩下降是季节性波动，还是内部调整所致

4. 从商户视角和用户视角分析，是商户问题还是用户问题，

- 如果是商户问题是哪个区域的商户问题，是不是集中在某个区域，因为竞争对手产生的影响或是天气等客观因素，还是内部运营问题。

- 或者是否集中在某群用户，比如老客流失的很多，或者这是个处于成长期的平台，新客很重要但新客增长开始放缓了。针对不同的客群，找不同的部门，是推广没做好，还是会员部没做好。

## 非技术题

### 1. 你的缺点是什么

考察点 - HR 想从应聘者身上获取的信息：

- 职业技能相关的硬性要求：专业技能过关、性格开朗
- 候选人自己对自己的评价
  - 管理能力不足 - 有想做管理者的想法
  - 没有什么缺点 - 对自己没有什么要求，目标也不够清晰

回答方法：

一横一纵回答法

正面表达：我觉得我对时间管理的能力还需加强
